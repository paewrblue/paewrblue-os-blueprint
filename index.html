⊕ Zoom-in — The Knowledge Loop: beliefs, narrators, edges (added 2026-07)

Research that only lives in dossiers decays. The knowledge loop compounds it: every serious document is distilled into belief files holding dated, falsifiable claims; every recurring author is graded; every repeatable insight becomes a codified edge. The mosaic now outweighs raw fundamentals in the conviction rubric.

DISTILL

/digest → beliefs/*.md

12-author Substack fan-out + Capital Wars corpus + primary filings → 3 belief files (global liquidity · optics/CPO · power semis), each a mechanism map + claims ledger.

→ 30+ dated claims · contested-questions register

SCORE

belief_checkpoints.py

Every claim carries a horizon; the checkpoint engine surfaces past-due claims in doctor.py so beliefs get scored right/wrong — not quietly forgotten.

→ claims scored at horizon · beliefs updated ≤30d

GRADE

expert_ledger + edge_log

12 narrators rated for reliability by domain; disagreements logged as spreads with arbiters. Repeatable patterns become numbered edges (e.g. derived-demand lag, commitments-attach).

→ 7 codified edges · narrator spreads w/ dated arbiters

⚒ Where to sharpen the machine — and why it moves the 2–3x goal

The system is already strong at collection (L1–L2) and synthesis (L4–L5). The biggest unrealized leverage is in the three things that convert all that activity into the actual outcome: reliability of the fastest signals, a feedback loop that proves the edge, and focus on the few names that can really return 2–3x. Honest read on the gaps, highest payoff first:

1 · Same-day procurement reliabilityHIGH · ALPHA
L1 SourcesL2 Automation
Gap
The fastest, most alpha-rich layer (official e-GP) is the most fragile: Turnstile-gated, ~10-min session TTL, and a dead session silently returns 0 rows that look like "no awards" rather than an error.
Fix
Hardened session manager — liveness probe before trusting an empty result, auto re-auth, queue + retry. The standing "solve Turnstile" task.
→ Goal
Reliable early read on Thai contract-driven revenue = entering SET names before the award reaches mainstream news = a bigger slice of the move.
2 · Feed health / heartbeatHIGH · DEFENSE
L2 AutomationL6 Outputs
Gap
13 launchd jobs run blind ("a plist on disk ≠ loaded"). A failed feed just leaves stale JSON behind — doctor.py checks data consistency, not whether a job actually ran.
Fix
A freshness/SLA health panel: last-run timestamp per feed, red when stale, LINE push on a missed run.
→ Goal
Decisions are only as good as the data. Catching a silent feed death stops you acting on stale prices or insider flow — capital preservation first.
3 · Automated thesis-breaker tripwiresHIGH · DEFENSE
L3 MemoryL4 Analysis
Gap
Half closed: macro regime tripwires (MOVE · T10Y2Y · SOFR−2y) now run live in the zone-alert engine with a Turbulence panel on the dashboard. Per-position fundamental breakers (margin print, utilization, customer-concentration events) in thesis_monitor.md are still checked manually.
Fix
Extend the tripwire engine from macro to per-position rules — encode each dossier's breaker as a machine-checkable condition and alert the moment it trips.
→ Goal
The #1 killer of compounding to 2–3x is riding a broken thesis down. Automated breakers force the exit on time.
4 · Conviction calibration loopMED · EDGE
L4 AnalysisL6 Outputs
Gap
Measurement now exists: decision_outcomes.py scores hit-rate + alpha per decision class vs SPX/TDEX. Still open: the result doesn't yet flow back into the sizing rubric automatically, and the mistake-log is still qualitative.
Fix
Close the loop: feed the scorecard's per-bucket results back into position_sizing_rules.md on a schedule.
→ Goal
Turns the journal into a measured edge. Better position sizing is the single biggest lever on getting to 2–3x.
5 · Idea triage funnelMED · FOCUS
L1→L4
Gap
~177 Thai + ~249 global names, but /analyze is deep and slow — analyst time is the bottleneck and there's no scored pre-filter pointing it at the best names.
Fix
A lightweight pre-screen (Minervini stage + valuation gap + catalyst proximity + insider/crowd signal) that ranks the backlog.
→ Goal
Quality over quantity — concentrate scarce deep-dive time on the few names that can really deliver 2–3x; stop spending cycles on the rest.
6 · One canonical live priceFOUNDATION
L3 MemoryL6 Outputs
Gap
Obsidian leg CLOSED (2026-07-04): obsidian_price_refresh.py writes currency-guarded live quotes into ~690 dossiers daily at 08:40 — the cockpit's "In Buy Zone NOW" is genuinely live. Remaining: zone-alerts and minervini_cache still fetch on their own cadence.

Fold the remaining two feeds onto the same fetcher so every surface reads one price.
→ Goal
The cardinal rule — never enter above the buy zone — only holds if every surface shows the same live price. Removes mis-timed or missed entries.
